# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Orbit is a RISC-V 64 (`rv64gc`) kernel written in Rust `no_std`, running on `qemu-system-riscv64` with the `virt` machine, 4 harts, 2 GiB RAM, an e1000 NIC, and M/S privilege modes. Target triple is `riscv64gc-unknown-none-elf`; toolchain is pinned in [rust-toolchain.toml](rust-toolchain.toml) to the nightly in the file (nightly-2026-02-21 at time of writing). `-Zbuild-std=core,alloc` is required ÔÇö do not run `cargo` without it.

There is no top-level `Cargo.toml`. `cargo` must be invoked from inside one of the individual crate directories. Each of `bl`, `kmain`, `umode`, `orbit-loader` has its own `.cargo/config.toml` carrying the `runner`, `rustflags`, and `build-std` settings.

## Build and run

Because each upper layer embeds the lower layer's ELF via `include_bytes!`, builds must happen in a specific order:

1. `cd orbit-loader && cargo build --release` ÔÇö compiles the TCP-listening loader that kmain boots into. `kmain` embeds `orbit-loader/target/riscv64gc-unknown-none-elf/release/orbit-loader` (see [kmain/src/kernel/mod.rs:85](kmain/src/kernel/mod.rs#L85)). Test payloads like `umode` are *not* compiled into the kernel ÔÇö they're built separately and sent over the wire via [orbit-loader/tools/send_payload.py](orbit-loader/tools/send_payload.py) once the guest is up.
2. `cd kmain && ./build.sh` ÔÇö builds the S-mode kernel as PIE with the required link flags (`-pie`, `-Bsymbolic`, `-znotext`, `--no-dynamic-linker`, `--pack-dyn-relocs=none`, `--export-dynamic`). Running `cargo build` directly in `kmain` will NOT produce a usable kernel ÔÇö use the script, which sets the `RUSTFLAGS` the kernel's self-relocation stub depends on. `bl` embeds `kmain/target/riscv64gc-unknown-none-elf/release/orbit` (see [bl/src/lib.rs:19](bl/src/lib.rs#L19)).
3. `cd bl && cargo run` ÔÇö builds the M-mode bootloader (binary name `launch`) and launches QEMU via the runner in [bl/.cargo/config.toml](bl/.cargo/config.toml). The runner also opens `-gdb tcp::1234` and passes `-S` (freeze on entry) so you can attach a debugger before `_start` runs. Guest port 7777 is forwarded to host 7777 (`hostfwd`) so `send_payload.py` can reach the loader.
4. (Optional, per iteration) `cd umode && cargo build && python3 orbit-loader/tools/send_payload.py umode/target/riscv64gc-unknown-none-elf/debug/umode` ÔÇö rebuild a test payload and ship it to the running guest. Avoids re-linking kmain+bl.

To debug a running QEMU: `./debug <crate> <exec>` ÔÇö e.g. `./debug bl launch` or `./debug kmain orbit`. This launches `rust-lldb` and issues `gdb-remote localhost:1234` against the chosen release ELF.

Unit tests live across the crates (notably `orbit-core/tests/`, plus `orbit-abi`, `net_channel`, `process`, `manager`, and `mem`) and run on the host via the `./test` script, which also drives a miri-hammer pass over the concurrency-sensitive suites. There are no lints configured.

### Building Orbit's std (forked rustc at [rust/](rust/))

Std-on-orbit binaries (currently only [hello-std/](hello-std/), eventually any user app that wants `std`) require a custom rustc fork that knows about the `riscv64gc-unknown-orbit` target. The fork lives at [rust/](rust/); its bootstrap config in [rust/bootstrap.toml](rust/bootstrap.toml) targets both the host and orbit. Build flow:

1. `cd rust && ./x build library --stage 1 --target riscv64gc-unknown-orbit` ÔÇö compiles `core`/`alloc`/`std` for orbit. Stage 1 is enough; stage 2 adds nothing relevant. Output lives under [rust/build/x86_64-unknown-linux-gnu/stage1/lib/rustlib/riscv64gc-unknown-orbit/lib/](rust/build/x86_64-unknown-linux-gnu/stage1/lib/rustlib/riscv64gc-unknown-orbit/lib/).
2. The toolchain is rustup-linked as `orbit-stage1` (run once: `rustup toolchain link orbit-stage1 /home/maya/projects/orbit/rust/build/x86_64-unknown-linux-gnu/stage1`).
3. `cd hello-std && cargo +orbit-stage1 build` ÔÇö builds against the freshly-built std. The `+orbit-stage1` selects the linked toolchain; without it cargo defaults to the project nightly which has no `riscv64gc-unknown-orbit` spec. `[build] target = "riscv64gc-unknown-orbit"` in `hello-std/.cargo/config.toml` pins the triple.

Whenever a `library/std/src/sys/...` file changes (PAL surface ÔÇö fs, net, thread, alloc, futex, ÔÇŽ) you must rerun step 1 before rebuilding the consumer. Skipping this is silent: cargo reuses the stale rlib in the toolchain sysroot and the new code never lands.

The `riscv64gc-unknown-orbit` target spec is built into the fork ([rust/compiler/rustc_target/src/spec/base/orbit.rs](rust/compiler/rustc_target/src/spec/base/orbit.rs)). It pins `linker = "rust-lld"`, static reloc model, no PIE, no eh_frame_header. orbit-abi's `rustc-dep-of-std` feature wires `core` to `rustc-std-workspace-core` so the kernel's pure-Rust types can be compiled into std.

For an as-built map of the orbit-side std PAL ÔÇö what's wired, what's stubbed, file paths, build-loop gotchas, and the priority order of remaining holes ÔÇö see [docs/dev/std-on-orbit.md](docs/dev/std-on-orbit.md).

External dependency: `smoltcp` is a path dependency at `../smoltcp` (sibling of this repo), not a crates.io version. If it is missing the workspace will fail to build `kmain`, `process`, and `net_channel`.

## Architecture

Three privilege levels run three separate ELF artifacts, each with its own linker script and `.cargo/config.toml`:

- **M-mode bootloader ÔÇö [bl/](bl/)** (binary `launch`, linked at `0x80000000` per [bl/memory.x](bl/memory.x)). Hart 0 (`kinit`) initializes the UART from the DTB, sets up PMP, installs the M-mode trap vector, parses the embedded kernel ELF (`KERNEL_ELF`), copies its `PT_LOAD` segments into physical RAM at `0x80000000 + 64 MiB` (see `VBASE` in [bl/src/lib.rs](bl/src/lib.rs)), publishes `bl::KMAIN_ENTRY`, and `sret`s into S-mode with a **bare satp** ÔÇö kmain's `_start` builds its own trampoline page tables, so bl no longer hand-sets an identity map. Secondary harts spin in `kinit_hart` until that wake, then `sret` into kmain's `_start`; per-hart context assignment (satp, `sscratch`, `sp`) now happens in S-mode via `SECONDARY_GO` / `KDMAP_BIAS_BOOT`, not over an M-mode `ecall`. Trap frames for all harts live at `0x80800000` (one per hart) so the M-mode trap handler can route interrupts while paging is still off. The M-mode stack region is split per-hart into a boot half and a trap half ÔÇö `m_trap_vector` switches `sp` into the hart's trap slot before calling the Rust handler, so trap spills don't clobber the boot stack.

- **S-mode kernel ÔÇö [kmain/](kmain/)** (binary `orbit`). Built as a fully relocatable PIE. Linked at low VA `0x1000`; an early trampoline in the naked `_start` in [kmain/src/bin/orbit.rs](kmain/src/bin/orbit.rs) builds a temporary Sv48 table (identity for RAM/MMIO plus `KTEXT_NOMINAL Ôćĺ load_addr` and a `KDMAP_NOMINAL Ôćĺ RAM` direct-map), `csrw satp`, and jumps into the high-half VA. Post-jump `post_trampoline_entry` applies `R_RISCV_RELATIVE` with `slide = ktext_base - LINK_BASE` before any Rust code touches a relocated global, then tail-calls `rust_main`. `rust_main` initializes the linked-list heap (`KHEAP` at its KDMAP VA), frame allocators for page tables (ktables) and general-purpose kernel pages (kpages) ÔÇö both returning KDMAP VAs via `add_frame_with_va_base` ÔÇö installs `OrbitLogger`/`OrbitSubscriber`, builds the final Sv48 table via `map_kernel_self` (KTEXT / KDMAP / KMMIO, no identity left), allocates per-hart `HartContext` structs, and `sret`s into `k_smpstart`. `k_smpstart` re-`init_serial`s at `kmmio_uart()` before any print, then sets `SECONDARY_GO[hart]` and wakes harts 1..N via `supervisor_wake_hart` (ACLINT SSWI). Every hart then enters `k_harthello` Ôćĺ `k_hart_loop`, greedily contending for the scheduler lock (`SchedGuard::try_with`) to run the manager pass when no other hart holds it. See [kmain/src/kernel/memmap.rs](kmain/src/kernel/memmap.rs) for layout atomics, constants, `KernelLayout`, and `RootTable` helpers.

- **U-mode boot process ÔÇö [orbit-loader/](orbit-loader/)** linked at `0x2_2000_0000` (`USER_TEXT_BASE`). The single process kmain spawns from its embedded ELF in `k_smpstart`. Listens on TCP :7777 via a NetChannel in listen mode; each incoming connection delivers a framed CBOR payload (`[u32 LE len][u32 LE !len][cbor {0: elf, 1: name, ÔÇŽ}]`, with optional keys for affinity and argv) that the loader hands to `create_process_v2` (syscall 4105, bytes mode, stamping uid/gid 1000). This replaces a build-time `include_bytes!` of a hardcoded sample; rebuilding a test binary no longer requires rebuilding kmain or bl.

- **U-mode sample ÔÇö [umode/](umode/)** linked at `0x2_2000_0000` (same `USER_TEXT_BASE` ÔÇö only one user ELF occupies that VA at a time today). A hardcoded demo ([umode/src/main.rs](umode/src/main.rs)) that prints to serial, sleeps, `mmap`s a shared region, registers a NetChannel, and opens a TCP connection. No longer embedded in kmain ÔÇö now shipped to a running guest via `orbit-loader/tools/send_payload.py`.

### U-mode VA layout

Sv48 low half is 128 TiB. The user-mappable space is split between kernel-managed regions (stacks, ELF ÔÇö installed at process creation, never via `mmap`) and user-controllable regions (priv heap, shared mmap/NetChannels). Constants in [orbit-abi/src/layout.rs](orbit-abi/src/layout.rs):

- `0..0x20_0000` ÔÇö null guard (2 MiB, megapage-aligned).
- `UPROC_STACK_BASE = 0x1000_0000` ÔÇö 256 per-thread slots, 32 MiB stride, 8 GiB total. Each slot holds `[stack at slot bottom (ÔëĄ 28 MiB)] [unmapped gap] [TLS reservation 2 MiB] [guard 2 MiB at slot top]`. Stack grows down; overflow falls into the previous slot's guard (or the unmapped span below `UPROC_STACK_BASE` for slot 0). Kernel-mapped at thread creation. See [docs/dev/user-thread-region.md](docs/dev/user-thread-region.md).
- `USER_TEXT_BASE = 0x2_2000_0000` ÔÇö ELF image. Kernel-mapped at process creation.
- `UPROC_PRIV_BASE = 0x3_0000_0000 .. UPROC_PRIV_END = 0x4000_0000_0000` ÔÇö user-controlled private range (~64 TiB). `mmap(share_with_kernel=false)` and orbit-rt's `#[global_allocator]` claim from here.
- `UPROC_SHARED_BASE = 0x4000_0000_0000 .. UPROC_SHARED_END = 0x7E00_0000_0000` ÔÇö user-controlled shared range (62 TiB). `mmap(share_with_kernel=true)` and `create_netch` (NetChannels) claim from here. orbit-rt's shared-VA region allocator (`SharedVa`) hands out from here with its own VA cursor.
- `USER_TRAP_FRAME_BASE = 0x7E00_0000_0000` ÔÇö kernel-private per-thread TrapFrames (S-only, no U bit).

The syscall layer (`orbit-core::syscall`) enforces the priv/shared split at the boundary: `mmap_req` rejects a private mmap aimed at a shared VA (and vice versa), `nc_create_req` rejects any VA outside the shared range, and the buffer-pointer syscalls (`serial_print`, `console_write`, `read_stdin`, `create_process`'s elf_ptr) accept anywhere user-mappable but reject kernel half / null guard / overflow via `user_range_ok`.

### Syscall ABI

`ecall` with syscall number in `a0`, args in `a1..a5` (most syscalls use `a1..a4`; `fb_present` uses five). Dispatched by the `cause == 8` arm of `s_trap` in [kmain/src/bin/orbit.rs](kmain/src/bin/orbit.rs). Canonical list in [orbit-abi/src/syscall.rs](orbit-abi/src/syscall.rs):

- `0` ÔÇö exit (noreturn)
- `1` ÔÇö serial_print(ptr, len)
- `2` ÔÇö sleep_ms(ms)
- `3` ÔÇö console_write(ptr, len)
- `4` ÔÇö read_stdin(ptr, len, flags)
- `12` ÔÇö read_key_event(buf, count, flags, timeout_ms) Ôćĺ n | -errno ÔÇö drain up to `count` 16-byte `KeyEvent`s from the calling process's structured-event ring. Companion to `read_stdin`: same producer ([kernel/input.rs](kmain/src/kernel/input.rs) `dispatch`) but exposes `KeyEvent { code, modifiers, kind }` (orbit-abi::input) before lossy encoding to UTF-8 + ANSI. Park shape mirrors `ch_yield`'s ms_sleep + wake_override pattern: caller stamps its tid into `ProcessKeyEvents.parked_tid`, sets `thread.wake_time = now + timeout_ms` (or `usize::MAX` for `READ_KEY_EVENT_INDEFINITE`), and yields `Suspended` via `YieldRetry`. `input::dispatch`'s producer side does `parked_tid.swap(0)` after the ring push, then `WAKE_QUEUE.push(WakeEvent::InputTid(tid))`; the manager's `drain_wakes` ORs `wake_reason::INPUT_IO` into the parked thread's `wake_override` and eagerly promotes Suspended Ôćĺ Ready. `flags & READ_KEY_EVENT_NONBLOCK` overrides timeout and returns `EAGAIN` on empty. `timeout_ms == 0` (no NONBLOCK) is a peek that returns 0 synchronously when empty. `EBUSY` if another tid is already parked. Per-process ring = [process::ProcessKeyEvents](process/src/key_events.rs); registry = [kmain/src/kernel/key_events.rs](kmain/src/kernel/key_events.rs); wake_reason bit added to [process/src/lib.rs](process/src/lib.rs) `wake_reason::INPUT_IO`; new `WakeEvent::InputTid(tid)` variant + drain_wakes arm in [kmain/src/kernel/mod.rs](kmain/src/kernel/mod.rs).
- `10` ÔÇö get_realtime() Ôćĺ (secs, nsec) ÔÇö wall-clock since UNIX epoch via Goldfish RTC at PA `0x101000` (driver: [kmain/src/drivers/goldfish_rtc.rs](kmain/src/drivers/goldfish_rtc.rs)). Two-register return; `nsec Ôłł [0, 999_999_999]`. Not monotonic ÔÇö for intervals use `get_micros` (8). Backs `SystemTime::now()` and `FileAttr::{modified,accessed,created}` in std PAL.
- `4096` ÔÇö mmap(vaddr, len, perms, share_with_kernel) ÔÇö vaddr must be in `UPROC_PRIV_BASE..UPROC_PRIV_END` when `share_with_kernel=false`, in `UPROC_SHARED_BASE..UPROC_SHARED_END` when `=true`
- `4097` ÔÇö create_netch(vaddr_hint, region_size, sock_type) Ôćĺ (user_va, fd) ÔÇö vaddr_hint must be in `UPROC_SHARED_BASE..UPROC_SHARED_END`
- `4098` ÔÇö close_handle(fd)
- `4099` ÔÇö create_process(elf_ptr, elf_len) Ôćĺ pid
- `4105` ÔÇö create_process_v2(args: *const CreateProcessV2Args) Ôćĺ pid ÔÇö role-aware spawn with explicit perms narrowing, optional cwd/argv/envp install, optional stdout capture, optional identity stamping, optional detach (`args.flags & CreateProcessV2Args::DETACH`: parent skips both `exit_waiter` notify and the `dead_children` insert when the child exits ÔÇö used by orbit-loader so its long-lived parent record doesn't accumulate per-payload exit-code entries that would otherwise grow the kernel-heap-backed `BTreeMap<u16, i32>` indefinitely under stress), and a two-front-door spawn source. **Spawn source**: `spawn_path_vaddr/spawn_path_len != 0` selects path mode ÔÇö kernel does `fs.open + vaccess(X)` against the caller's effective creds, then reads the ELF off disk through the page cache ÔÇö `PendingWork::CacheFill` completions feed a manager-side `SpawnInProgress` state machine (`issue_next_spawn_page` Ôćĺ the canonical `install_spawn` helper). `spawn_path_vaddr == 0` selects bytes mode ÔÇö kernel reads `elf_vaddr[..elf_len]` from caller user memory; restricted to `role::LOADER` because there's no file to check the X bit against. Non-LOADER bytes-mode Ôćĺ `-EPERM`. orbit-loader (LOADER) uses bytes mode for network-payload spawns; console/std PAL/etc. use path mode. **Identity stamping**: `setuid_uid` / `setuid_gid` (`-1` = inherit, `0..=u32::MAX` = stamp on all three triplet slots), `setlogin_vaddr/len` (`0` = inherit), `groups_vaddr/count` (`0` = inherit; non-zero installs the listed supp groups, capped at `process::NGROUPS_MAX`). Stamping any non-inherit value requires the *parent* to be running with `role::LOADER` ÔÇö other callers get `-EPERM`. orbit-loader stamps `uid=1000`/`gid=1000` on every payload it spawns; that's the only non-root identity path until a real login pipeline lands.
- `4107` ÔÇö chdir(path_ptr, path_len) Ôćĺ 0 ÔÇö replace caller's cwd with the absolute UTF-8 path; kernel validates the dir exists in the active fs before mutating
- `4108` ÔÇö getcwd(buf_ptr, buf_len) Ôćĺ bytes_written ÔÇö copy the caller's cwd (no NUL) into the user buffer; ERANGE if too small. Children inherit the parent's cwd at spawn (`CreateProcessV2Args` carries an optional `cwd_vaddr`/`cwd_len` override; legacy `CREATE_PROCESS` / `CREATE_PROCESS_EX` always inherit). Relative-path fs syscalls (`fs_open`, `fs_stat`) are prefixed by `Process.cwd` kernel-side.
- `4109` ÔÇö getuid() Ôćĺ uid ÔÇö POSIX `getuid(2)`. Real uid of the calling process. Reads the per-thread `uid` credential snapshot (an atomic, refreshed from `Process.uid` at thread creation and on `setuid`) without locking. uid/gid feed the `vaccess` check `fs_open` runs against inode ownership; roles + permissions own the rest of authorization. orbit-loader stamps payloads with uid 1000.
- `4110` ÔÇö geteuid() Ôćĺ euid ÔÇö POSIX `geteuid(2)`. Effective uid. Split from `getuid` (rather than bundled into `(uid << 32) | euid`) because uids Ôëą 0x8000_0000 would shift into the high bit of the `isize` return and be misread as a negative errno.
- `4111` ÔÇö getgid() Ôćĺ gid ÔÇö POSIX `getgid(2)`. Real gid; mirror of `getuid`.
- `4112` ÔÇö getegid() Ôćĺ egid ÔÇö POSIX `getegid(2)`. Effective gid; mirror of `geteuid`.
- `4113` ÔÇö getgroups(buf_ptr, count) Ôćĺ count | -ERANGE ÔÇö POSIX `getgroups(2)`. Copy the caller's supplementary group list into the user buffer (one `u32` per slot). `count == 0` returns the current group count without writing (POSIX sizing call). Capped at `process::NGROUPS_MAX = 16`.
- `4114` ÔÇö getlogin(buf_ptr, buf_len) Ôćĺ bytes | -errno ÔÇö POSIX `getlogin_r(3)`. Copy the calling process's session login name (no NUL terminator) into the user buffer. `ENOENT` if no login name has been installed (initial process state).
- `4115` ÔÇö setuid(uid) Ôćĺ 0 | -EPERM ÔÇö POSIX `setuid(2)`. `euid == 0` stamps `uid` on all three triplet slots (privilege drop). `euid != 0` sets only euid, IFF `uid Ôłł {ruid, suid}` (POSIX privilege-toggle). Manager round-trip (`PendingWork::SetUid`): the manager mutates the triplet and refreshes per-thread credential snapshots so sibling threads' `getuid` reads observe the new identity. Gated on `class::PROC_CRED` (separate from PROC_LIFE so a daemon can pledge it away after privsep startup).
- `4116` ÔÇö setgid(gid) Ôćĺ 0 | -EPERM ÔÇö POSIX `setgid(2)`. Gid mirror of `setuid` (POSIX still keys the privilege slot off `euid == 0`, not gid).
- `4117` ÔÇö setgroups(buf_ptr, count) Ôćĺ 0 | -errno ÔÇö POSIX `setgroups(2)`. Replace the caller's supplementary group list. Requires `euid == 0`; capped at `process::NGROUPS_MAX = 16`. `count == 0` is legal (empties the list). Gated on `class::PROC_CRED`.
- `4118` ÔÇö setlogin(name_ptr, name_len) Ôćĺ 0 | -errno ÔÇö POSIX `setlogin(2)`. Stamp the calling process's session login name (UTF-8, ÔëĄ 32 bytes, requires `euid == 0`). Gated on `class::PROC_CRED`.
- VFS access enforcement: `fs_open` runs `vaccess()` against the calling process's `(euid, egid, supplementary groups)` and the inode's `(st_uid, st_gid, st_mode)` after path resolution. OwnerÔćĺgroupÔćĺother check, first match wins; root (`euid == 0`) bypasses. Returns `EACCES` on the deny path. Pure rule logic + unit tests live in [orbit-abi/src/fs.rs](orbit-abi/src/fs.rs) (`vaccess` fn); the kmain adapter is `Orbit::vaccess_pid` in [kernel/mod.rs](kmain/src/kernel/mod.rs). Path-walk traversal (X on each parent dir) isn't modeled ÔÇö tarfs's `open(path)` is a single-pass internal walk; only the final inode is checked. `fs_stat` doesn't run vaccess (POSIX); `fs_read` doesn't re-check (perm checked at open time). Test fixture: `/etc/secret` is staged as `0o600 root:root` in tarfs via [tools/build-disk.sh](tools/build-disk.sh)'s second-pass `--owner=0 --group=0` for the `etc/` subtree.
- `6004` ÔÇö fs_seek(fd, offset, whence) Ôćĺ new_offset ÔÇö POSIX-shaped: `SEEK_SET = 0`, `SEEK_CUR = 1`, `SEEK_END = 2`. Mutates the per-fd `OpenFile.offset` only; rejects directory fds (`EBADF`) and resolved-negative offsets (`EINVAL`).
- `6005` ÔÇö fs_fstat(fd, &mut Stat) Ôćĺ 0 ÔÇö fill `*stat` with metadata for the file backing `fd`. Mirror of `fs_stat` keyed on an open fd.
- `7000` ÔÇö fb_query(&mut FbInfo) Ôćĺ 0 ÔÇö fill `*info` with the active framebuffer dims and pixel format. Stable for the system's lifetime in v1; cache the result. Rejects buffers that straddle a page (`EINVAL`) or don't translate (`EFAULT`).
- `7001` ÔÇö fb_surface_create(w, h, format) Ôćĺ (handle, user_va) ÔÇö allocate a `kernel_pages`-backed pixel surface of `w*h*bpp` bytes (rounded up to `PAGE_SIZE`), map it user-writable in the calling process's shared range, and register a per-process [SURFACE_TABLE](kmain/src/kernel/surface.rs) entry. `format = 1` (BGRA8888) is the only value v1 accepts. Backing frame ownership lives on the surface entry, not `Process.heap_pages`; `dealloc_process` walks `SURFACE_TABLE.unregister(pid)` to recover them. Two-register return (`signal_pair`) ÔÇö handle in `a0`, user_va in `a1`.
- `7002` ÔÇö fb_surface_destroy(handle) Ôćĺ 0 ÔÇö manager removes the entry, `unmap_range`s the user VA, and frees the backing frame. `EBADF` for unknown handles.
- `7003` ÔÇö fb_present(handle, x, y, w, h) Ôćĺ 0 ÔÇö sync. Look up the surface, validate the rect against its dims, push a `Cmd::PresentSurface` carrying the snapshot (kdmap KVA, dims, format, rect) onto k_gpu's `CONSOLE_RING`. The compositor blits the rect via the surface's KDMAP alias and unions damage rects into a single `transfer_to_host_2d` per drain pass. First `fb_present` from a process flips that source to surface mode in [kmain/src/drivers/display.rs](kmain/src/drivers/display.rs); `console_write` bytes still accumulate in scrollback but stop driving repaints until the surface is destroyed.

### Supporting crates (all under the same workspace)

- [mmu/](mmu/) ÔÇö Sv48 page tables, `PagePermissions`, `MappingConfig`, `id_map_range`, `map_address_range`, `map_va_range`, `virt_to_phys`. Walkers take `&RootTable<'_>`, which pairs a `&PageTable` with a PAÔćĺVA bias so they can follow PPNs (always physical) back into the supervisor's KDMAP view. bl/early-trampoline use `RootTable::identity(...)` (bias 0); kmain uses `memmap::kernel_root(...)` / `kernel_root_from_pa(...)` (bias `= kdmap_base - ram_phys_base`). `alloc` feature wires it up against `alloc::Vec`-backed page pools.
- [mem/](mem/) ÔÇö `FrameAllocator`, `round_u64_up`/`down` helpers, `prev_power_of_two`.
- [device/](device/) ÔÇö `HartContext` (cache-line-aligned, field offsets are load-bearing ÔÇö consumed by `asm/trap.S` and `boot.S`), `TrapFrame`, `Stack` (2 MiB), `SysInfo`, DTB walkers (`find_serial_port`, `find_ram`). The trap assembly hard-codes offsets into `HartContext`; changing field order requires updating the `.S` files.
- [process/](process/) ÔÇö `Thread`, `Process`, `ThreadState`, the typestate capability layer ([process/src/cap.rs](process/src/cap.rs)) that gates all `Thread` mutation, the scheduler lock (`SchedGuard` in [process/src/sched_lock.rs](process/src/sched_lock.rs), historically called `MANAGER_LOCK`), and the `CompletionHandle` / `AckCounter` slot primitives. A blocking syscall parks the thread `Blocking` and pushes a tid-carrying `PendingWork` variant onto the `MANAGER_WORK` thingbuf; whichever hart next holds the scheduler lock runs `Orbit::drain_pending_work`, executes the matching arm, and resumes the parked thread via `publish_pending_for_tid` (writes up to 4 a-regs into the parked frame, then pushes `WakeEvent::Tid`). New blocking syscalls add a variant to [orbit-core/src/pending_work.rs](orbit-core/src/pending_work.rs) and a handler arm in `drain_pending_work` ÔÇö no per-reason field on `Thread`, and no `CompletionHandle` is carried in the work item. Syscall handlers must NOT touch manager-owned state (`process_handles`, `Process` cwd/cred fields, the `processes` map) from the trap path ÔÇö even read-only, since the manager's BTreeMap mutations on another hart race the walk. Route through `PendingWork` instead; per-thread atomic snapshots (`uid` etc.) are the sanctioned lock-free fast path. No spinlocks in trap handlers ÔÇö a trap/exception while holding one re-creates the kptr-long-jump deadlock the scheduler lock is documented to avoid.
- [net_channel/](net_channel/) ÔÇö shared-memory SPSC queues between user programs and the kernel networking thread. `kernel` feature enables the kernel-side `update_tcp` methods that drive smoltcp sockets. Layout of `NetChannel` / `NetChannelCurrent` / `NetChannelQueue` is part of the user/kernel ABI ÔÇö do not change field order casually.
- [serial/](serial/) ÔÇö `MpUart` wrapping `ns16550a::Uart` with a spinlock; exposes `println!` macro and a global init.
- [kmain/src/drivers/e1000.rs](kmain/src/drivers/e1000.rs) ÔÇö PCI-discovered e1000 NIC; ring buffers fed by DMA from kernel pages, integrated with smoltcp via `k_net` (a dedicated kernel thread).
- **Page-cache-backed spawn / fs reads** ÔÇö path-mode `create_process_v2` and `fs_read` pull bytes through the page cache ([kmain/src/kernel/page_cache.rs](kmain/src/kernel/page_cache.rs)): the virtio-blk IRQ posts `PendingWork::CacheFill` completions, which the manager folds into the relevant waiter ÔÇö for spawns, a `SpawnInProgress` state machine (`issue_next_spawn_page` Ôćĺ `install_spawn`); for reads, the parked `fs_read` caller.
- **Kernel threads** ÔÇö `k_net` (smoltcp/e1000 pump), `k_gpu` (virtio-gpu compositor; drains `CONSOLE_RING`), and `k_serial` (exclusive UART owner; logs/traces are fed to it via `SERIAL_RING`). Each is spawned at boot and parks via `kthread_park` between wakeups.

### SMP and scheduling

`HartContext` is the per-hart state structure. `sscratch` always points to the current hart's `HartContext` (as a KDMAP VA under the kernel satp). Cross-hart wakeups go via ACLINT SSWI at `kmmio_sswi()` (physical `0x02F00000`; one `u32` per hart ÔÇö `write_sswi`/`supervisor_wake_hart`). Timer interrupts use Sstc (`stimecmp` at CSR `0x14D`). The scheduler is greedy and unpinned: every hart runs `k_hart_loop` and tries to take the scheduler lock (`SchedGuard::try_with`); whichever hart holds it runs the manager pass (`assign_threads` etc.) and IPIs idle harts. Harts receive S-mode software interrupts in `s_trap` (async cause `1`) and context-switch via `check_context_and_switch`. `s_trap_vector` stays on whichever `satp` triggered the trap; syscall handlers that reach user memory walk the user PT manually and go through the KDMAP alias (`memmap::user_va_to_kdmap`), with a `UserAccess` SUM guard covering direct user reads.
